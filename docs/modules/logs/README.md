# README.md

## Module Working Records

## Purpose

This directory will store working logs for module-level design as the module docs move beyond placeholder status.

## Intended Use

Use module logs to capture:

- options considered for a module
- assumptions and revisions
- interface or contract decisions
- deferred questions
- changes in direction over time

## Relationship To Module Docs

- `docs/modules/*.md` should hold the stable module view
- `docs/modules/logs/` should hold working notes and design history

## Status

Placeholder structure only. Module working logs should be added when module design sessions become substantive.
