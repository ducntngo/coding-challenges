# README.md

## AI Usage Diary

## Purpose

This directory stores working-log style records of how AI was used throughout the project. The logs are intended to map closely to commits or small groups of related commits.

## Why This Exists

The coding challenge explicitly requires documenting AI collaboration. These diary entries provide a low-friction way to capture that evidence continuously instead of reconstructing it at the end.

## Logging Rule

Add a new entry for each meaningful commit or small batch of tightly related commits where AI materially contributed to:

- design
- implementation
- refactoring
- debugging
- test generation
- documentation

In addition, every pull request must include diary coverage for the AI-assisted work contained in that PR. One entry may cover the full PR if the work is cohesive.

## Suggested Naming

Use sortable file names:

- `YYYY-MM-DD-01-short-topic.md`
- `YYYY-MM-DD-02-short-topic.md`

## Entry Template

Each entry should cover:

- date
- related commit(s) or branch
- task summary
- AI tools used
- prompt or interaction summary
- outputs influenced by AI
- human verification and refinements
- risks or follow-up items

## Baseline Format

Use `2026-03-27-01-doc-scaffolding.md` as the baseline style. The content should also satisfy the challenge README expectation to document:

- which AI tools were used
- what tasks AI helped with
- the nature of prompts or interactions
- what verification, testing, debugging, or refinement was performed

## Relationship To Final Submission

These logs are working notes. Later, they will be condensed into the final AI collaboration document for the submission package.
