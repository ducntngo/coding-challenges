# README.md

## Architecture Working Records

## Purpose

This directory stores working logs for architecture exploration, architecture decision changes, and major design-boundary reviews.

## Intended Use

Use architecture logs to capture:

- problem framing and scope decisions
- architecture options considered
- major tradeoffs and rationale
- explicit assumptions and revisions
- open questions and deferred risks
- resume context for later architecture work

## Relationship To Architecture Docs

- `docs/ARCHITECTURE_PRINCIPLES.md` should hold the stable architecture baseline
- `docs/architecture/TRADEOFFS.md` should hold canonical tradeoff summaries
- `docs/architecture/logs/` should hold working notes and architecture decision history

## Template

Use `docs/architecture/logs/TEMPLATE.md` as the default starting point for new architecture working records.
